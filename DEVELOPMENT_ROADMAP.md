## 决策事项

### 技术选型（单机版）

| 功能 | 选型 | 说明 |
|------|------|------|
| 缓存 | 本地内存 | 单机足够，无需Redis |
| 数据库 | SQLite | 单机首选，零配置 |
| 消息队列 | 内存队列 | 无需外部依赖 |
| 前端状态 | Zustand | 保持现状 |
| 部署方式 | Docker Compose | 单机编排 |
| 编辑器 | Monaco Editor | VS Code内核 |
| ML框架 | scikit-learn | 轻量易用 |

### 需要确认的业务需求

1. **是否需要分账户权限管理？**（多交易员场景）
2. **是否需要接入多个交易所？**（同时交易CTP、恒生等）
3. **数据保留策略？**（历史数据保留多久）
4. **合规要求？**（是否需要审计日志、数据加密）

## 风险评估

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| vnpy 引擎稳定性 | 中 | 高 | 完善异常捕获、自动重启 |
| ML模型过拟合 | 高 | 中 | 交叉验证、 walk-forward测试 |
| 策略编辑器安全风险 | 中 | 高 | 代码沙箱、禁止危险操作 |
